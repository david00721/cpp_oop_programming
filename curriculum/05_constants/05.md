# Konstansok és a `const` kulcsszó

A `const` kulcsszóval megakadályozhatjuk, hogy egy változó (vagy objektum) értékét módosítsák a létrehozása után. Ezt hívjuk "read-only" (csak olvasható) állapotnak.

# Konstansok alapjai

Ha egyszer adtál neki egy értéket, az többé nem változhat.

```cpp
const int maxSpeed = 120;
// maxSpeed = 150; // HIBA: egy konstans értékét nem lehet felülírni

```

## Mutatók és konstansok

### 1. Mutató egy konstans értékre (Pointer to const)

**A metafora:** Képzelj el egy múzeumi idegenvezetőt (a pointer). A vezető rámutathat egy festményre, aztán átmehet és rámutathat egy szoborra. Viszont **semmit sem firkálhat össze** – az alkotások (az értékek) érinthetetlenek.

Itt maga a mutató flexibilis, de az érték, amire épp mutat, védett.
*(Szakmai érdekesség: a `const int*` és az `int const*` teljesen ugyanazt jelenti a C++-ban.)*

```cpp
int appleCount = 5;
int orangeCount = 10;

// Olvasd jobbról balra: A 'ptr' egy mutató (*), ami egy egész számra mutat, ami konstans (const).
const int* ptr = &appleCount; 

// Mit szabad?
ptr = &orangeCount;  // OK: A mutatót átirányíthatjuk egy másik változóra.

// Mit NEM szabad?
// *ptr = 20;        // HIBA: A mutatott értéket a pointeren keresztül TILOS módosítani.

```

### 2. Konstans mutató (Const pointer)

**A metafora:** Képzelj el egy falra szerelt biztonsági kamerát (a pointer). A kamera **be van betonozva**, mindig pontosan ugyanarra az egy ajtóra kell néznie, nem fordíthatod el. Viszont maga az ajtó (az érték) nyitva vagy zárva is lehet, sőt, akár át is festhetik – a kamera ezen nem változtat, csak fixen figyeli.

Itt az érték változhat, de a mutató örökre ahhoz a memóriacímhez van láncolva, amivel inicializáltad.

```cpp
int playerHealth = 100;
int enemyHealth = 50;

// Olvasd jobbról balra: A 'ptr' egy konstans (const) mutató (*), ami egy egész számra (int) mutat.
int* const ptr = &playerHealth;

// Mit szabad?
*ptr = 80;           // OK: A mutatott érték (a játékos élete) módosítható.

// Mit NEM szabad?
// ptr = &enemyHealth; // HIBA: A mutató "be van betonozva", nem mutathat a rögzítés után másra.

```

### 3. Konstans mutató egy konstansra (Const pointer to const)

Ezt a verziót ritkábban említik az alapoknál, pedig ez teszi teljessé a képet!

**A metafora:** Képzelj el egy bebetonozott műemlékvédelmi táblát, ami egy érinthetetlen történelmi ereklyére mutat. Sem a táblát nem viheted máshova, sem az ereklyét nem kalapálhatod szét. Minden maximálisan védett.

```cpp
int maxLevel = 99;
int maxScore = 9999;

// Olvasd jobbról balra: A 'ptr' egy konstans (const) mutató (*), ami egy konstans (const) egész számra (int) mutat.
const int* const ptr = &maxLevel;

// Mit szabad?
// Csak kiolvasni az értéket: int currentLimit = *ptr;

// Mit NEM szabad?
// *ptr = 100;       // HIBA: Az érték védett.
// ptr = &maxScore;  // HIBA: A mutató be van betonozva.

```

---

### Összefoglaló

Hogy vizuálisan is könnyű legyen megjegyezni, itt egy gyors táblázat:

| Szintaktika | Átirányítható a mutató? | Módosítható az érték? |
| --- | --- | --- |
| `const int* p` | Igen | Nem |
| `int* const p` | Nem | Igen |
| `const int* const p` | Nem | Nem |

> **💡 Tipp:** Amikor függvényeket írsz, nagyon gyakran fogod használni a `const` kulcsszót a paramétereknél (pl. `void printArray(const int* arr)`), hogy garantáld: a függvény csak beolvassa az adatokat, de véletlenül sem rontja el az eredeti tömböt.

# 2. Konstansok az Osztályokban (Classes)


---

### 1. Konstans tagváltozók (Const member variables)

**A metafora:** Képzeld el egy autó alvázszámát (VIN). Amikor az autó a gyárban lekerül a futószalagról (ez a konstruktor), ráütik az alvázszámot. Onnantól kezdve az autó élete végéig ez a szám változatlan marad. Sem a szerelő, sem a tulajdonos nem kaparhatja le, hogy átírja.

A C++-ban a konstans tagváltozóknál az a fő trükk, hogy mire a konstruktor kapcsos zárójelei `{ ... }` közé érsz, a változók már "megszülettek". Ezért nem adhatsz nekik ott értéket (hiszen egy konstanst csak a születése pillanatában lehet inicializálni). Erre való az **inicializáló lista**.

```cpp
class Employee {
private:
    const int employeeId; // "Alvázszám", sosem változik
    std::string name;     // A név viszont megváltozhat (pl. házasság)

public:
    // A kettőspont utáni rész az inicializáló lista. Itt dől el a konstans sorsa!
    Employee(int id, std::string empName) : employeeId(id), name(empName) {
        // employeeId = 999; // HIBA: Itt már túl késő lenne értéket adni egy konstansnak!
    }
};

```

### 2. Konstans tagfüggvények (Const member functions)

**A metafora:** Képzeld el a bankszámládat. Van egy "Egyenleg lekérdezése" gomb, és egy "Átutálás" gomb. Az "Egyenleg lekérdezése" egy `const` metódus: akármennyiszer nyomod meg, a pénzed (az objektum állapota) nem lesz kevesebb. Csak "read-only" (olvasási) jogot kapott.

Ezt nem csak illik odaírni a gettekhez, hanem **kötelező** is, ha stabil kódot akarsz írni. Ugyanis, ha van egy `const` objektumod, a fordító *csak és kizárólag* a `const` tagfüggvényeket engedi meghívni rajta!

```cpp
class BankAccount {
private:
    int balance;
public:
    BankAccount(int b) : balance(b) {}

    // Konstans metódus: Garantálja, hogy csak nézelődik, de nem nyúl a pénzhez.
    int getBalance() const { 
        // balance = 0; // HIBA: A fordító azonnal rácsap a kezedre.
        return balance; 
    }

    // Nem konstans metódus: Ez módosítja az állapotot.
    void deposit(int amount) {
        balance += amount;
    }
};

void printStatement(const BankAccount& account) {
    // Mivel az 'account' itt konstans referenciaként jött be, 
    // CSAK const metódusokat hívhatunk meg rajta!
    std::cout << account.getBalance(); // OK
    // account.deposit(100);           // HIBA: Egy konstans számlára nem fizethetsz be!
}

```

### 3. A `mutable` kulcsszó: A megengedett "kiskapu"

**A metafora:** Képzelj el egy múzeumi festményt a vitrinben (ez egy `const` objektum). A látogatók csak nézhetik (csak `const` metódusokat hívhatnak meg rajta), maga a festmény nem változik. Viszont a múzeumigazgató felszerelt egy rejtett kattintós számlálót a teremben, ami méri, hányan nézték meg a képet. Maga az alkotás érintetlen maradt (logikai konstanság), de a háttérben egy számláló (a `mutable` változó) mégis pörög.

A `mutable` pontosan erre való: olyan belső változókhoz, amik nem részei az objektum "logikai" állapotának (pl. számlálók, gyorsítótárazás/caching, mutex lockok).

```cpp
class HeavyCalculation {
private:
    int data;
    mutable bool isCached; // A const függvény is átírhatja!
    mutable int cachedResult;

public:
    HeavyCalculation(int d) : data(d), isCached(false), cachedResult(0) {}

    // Logikailag ez a függvény csak lekérdez, de a háttérben optimalizál.
    int getResult() const {
        if (!isCached) {
            cachedResult = data * 9999; // Lassan kiszámoljuk...
            isCached = true;            // OK, mert mutable!
        }
        return cachedResult;
    }
};

```

### 4. A `const_cast`: A vészkalapács

A C++-ban a típuskonverzióknak több fajtája van, a `const_cast` a legagresszívebb, ha konstansokról van szó.

**A metafora:** Olyan, mint a "Tűz esetén betörni!" feliratú üveg. Odaadnak neked egy zárt dobozt (const), de te feszítővassal mégis kinyitod.

**Hatalmas piros felkiáltójel:** Ha egy változó *eredetileg* `const`-ként született, és te `const_cast`-tal megpróbálod módosítani, az a C++-ban **Undefined Behavior (Nem definiált viselkedés)**. Bármi megtörténhet: a program összeomolhat, de akár látszólag működhet is hibásan. Általában csak akkor használjuk, ha egy régi, C-nyelven írt (Legacy) könyvtárral kell kommunikálni, ami paraméterként `char*`-ot kér, de mi tudjuk, hogy valójában nem fogja módosítani a mi `const char*` adatunkat.

```cpp
void legacyCFunction(int* ptr) {}

void myModernFunction(const int* myData) {
    // legacyCFunction(myData); // HIBA: const int*-ot nem adhatunk int*-nak.
    
    // "Bízom benne, lezúzom a const védelmet:"
    legacyCFunction(const_cast<int*>(myData)); // Így már átmegy a fordítón.
}

```
