# Hozzáférési Jogosultságok (Access Specifiers)

Eddig a `public:` kulcsszót használtuk, ami azt jelentette, hogy "mindenki láthat mindent". De a valóságban nem akarjuk, hogy bárki szabadon átírhassa az objektumaink belső adatait (például, hogy a kutya életkora -5 legyen).

C++-ban három fő jogosultsági szint van:

1. **`public` (Nyilvános):** Bárhonnan elérhető (az osztályon kívülről is).
2. **`private` (Privát):** Csak az osztályon **belülről** érhető el. (Ha nem írsz ki semmit, alapértelmezetten minden `private`!)
3. **`protected` (Védett):** Hasonló a priváthoz, de az öröklődésnél van szerepe (ezzel később foglalkozunk).

### Miért jó ez? (Encapsulation)

Ezt hívjuk **Enkapszulációnak** (bezárásnak). Olyan ez, mint egy bankautomata:

* **Public:** A gombok és a képernyő (amit használsz).
* **Private:** A pénzkazetta és a számítógép belseje (amihez nem nyúlhatsz közvetlenül).

### Példa

```cpp
class MyClass {
  public:    // Innentől minden nyilvános
    int x;   
  private:   // Innentől minden privát
    int y;   
};

int main() {
  MyClass myObj;
  myObj.x = 25;  // Ez MŰKÖDIK (public)
  // myObj.y = 50;  // Ez HIBÁT OKOZ (private) - "y is private"
  return 0;
}

```

# Encapsulation

Hogyan érjük el a privát adatokat mégis?
**Getter** (lekérdező) és **Setter** (beállító) metódusokkal! Ezek "kapuőrként" működnek: ellenőrizhetik az adatot, mielőtt beengedik.

Az enkapszuláció lényege, hogy elrejtjük az "érzékeny" adatokat a felhasználó elől. Ezt két lépésben érjük el:

1. A változókat (attribútumokat) **`private`**-ként deklaráljuk (ezt már megcsináltuk).
2. Publikus **`get`** (lekérdező) és **`set`** (beállító) metódusokat biztosítunk az adatok eléréséhez és módosításához.

### Miért jók a Getterek?

Elérhessük az osztály privát adattagjait.

### Miért jók a Setterek?

Miért nem hagyjuk csak publikusan a változót? Mert a **Setter** metódussal kontrollálhatjuk, hogy mi kerüljön az adatba. Például megakadályozhatjuk, hogy valaki negatív életkort állítson be.

### Példa

Itt egy példa, ahol a fizetést (`salary`) csak ellenőrzött módon lehet beállítani:

```cpp
class Employee {
  private:
    int salary; // Private attribute

  public:
    // Setter (Beállító)
    void setSalary(int s) {
      if (s >= 0) {       // Validation logic
        salary = s;
      } else {
        cout << "Error: Salary cannot be negative!" << endl;
      }
    }

    // Getter (Lekérdező)
    int getSalary() {
      return salary;
    }
};

int main() {
  Employee myObj;
  myObj.setSalary(50000); // Helyes beállítás
  cout << myObj.getSalary();
  return 0;
}


```

### A `const` függvények

Ha egy metódus után odaírjuk a `const` kulcsszót, azzal **megígérjük**, hogy ez a függvény **SOHA nem fogja módosítani** az osztály adatait.

**Mire jó?** Ez biztonságot ad. A `get` (lekérdező) metódusoknál kötelező elem a profi kódban, hiszen azoknak csak olvasniuk szabad, írniuk nem.

```cpp
class Example {
  private:
    int x;
  public:
    // Ez egy csak olvasható metódus
    int getX() const {
      // x = 5; // Ez HIBÁT okozna, mert const függvényben vagyunk!
      return x;
    }
};

```

### A return *this

Ez a parancs teszi lehetővé a **metódusok láncolását** (Method Chaining), ami sokkal tömörebbé és olvashatóbbá teheti a kódot.

**Hogyan működik?**

1. **`this`**: Ez egy mutató (pointer), ami mindig az aktuális objektum memóriacímét tárolja.
2. **`*this`**: A `*` operátorral "dereferáljuk" a mutatót, vagyis magát az **objektumot** érjük el általa.
3. **`return *this;`**: A függvény visszaadja magát az objektumot, amin éppen dolgoztunk.

**A megvalósítás:**
Ahhoz, hogy ez jól működjön, a metódus visszatérési típusának az osztály **referenciájának** (`Osztaly&`) kell lennie.

```cpp
class Teglalap {
    int szelesseg, magassag;
public:
    // Figyeld a visszatérési típust: Teglalap& (referencia)
    Teglalap& setSzelesseg(int sz) {
        this->szelesseg = sz;
        return *this; // Visszaadja a jelenlegi objektumot
    }

    Teglalap& setMagassag(int m) {
        this->magassag = m;
        return *this;
    }
};

int main() {
    Teglalap t;
    // Láncolás: egy sorban hívjuk meg a metódusokat
    t.setSzelesseg(10).setMagassag(20); 
}

```

### Gyakorló feladat

**Specifikáció (Követelmények):**

1. **Osztály létrehozása:**
* Neve legyen `Rectangle`.


2. **Adatok (Attribútumok):**
* Legyen két **privát** változója (egész számok): `width` (szélesség) és `height` (magasság).


3. **Konstruktor:**
* Legyen egy publikus konstruktora: `Rectangle(int w, int h)`.
* Állítsa be a kezdőértékeket.


4. **Enkapszuláció (Getterek és Setterek):**
* Írj **get** metódust mindkét változóhoz: `getWidth()`, `getHeight()`.
* Írj **set** metódust mindkét változóhoz: `setWidth(int w)`, `setHeight(int h)`.
* **Validáció:** A set metódusok (és a konstruktor is) csak akkor fogadják el az értéket, ha az **nagyobb, mint 0**. Ha 0 vagy negatív számot kapnak, írjanak ki hibaüzenetet (pl. *"Error: Value must be positive"*), és ne módosítsák az adatot (a konstruktor esetén állítsanak be egy alapértelmezett értéket, pl. 1-et).


5. **Számítási Metódusok:**
* Készíts egy `getArea()` metódust, ami visszaadja a téglalap területét ().
* Készíts egy `getPerimeter()` metódust, ami visszaadja a téglalap kerületét ().


6. **Destruktor:**
* A `~Rectangle` destruktor a program végén írja ki: *"Rectangle deleted."*


7. **Tesztelés (Main függvény):**
* Hozz létre egy példányt érvényes adatokkal.
* Próbáld meg átállítani az egyik oldalt egy negatív számra a setter segítségével.
* Írasd ki a végleges kerületet és területet.
