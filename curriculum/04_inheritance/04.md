# Öröklődés (Inheritance) C++-ban

C++-ban (és az OOP-ben általában) lehetőségünk van arra, hogy egy osztály átvegye egy másik osztály tulajdonságait és metódusait. Ezt hívjuk **öröklődésnek**.

Két fő szereplő van:

1. **Szülő osztály (Base Class / Parent):** Az az osztály, amelytől öröklünk.
2. **Gyerek osztály (Derived Class / Child):** Az az osztály, amely örökli a tulajdonságokat.

---

### Szintaxis

A gyerek osztály neve után kettőspontot `:` teszünk, majd megadjuk az **öröklődés módját** (általában `public`) és a szülő osztály nevét.

```cpp
// Base class (Szülő)
class Vehicle {
  public:
    string brand = "Ford";
    void honk() {
      cout << "Tuuutuuut! \n";
    }
};

// Derived class (Gyerek)
class Car : public Vehicle {
  public:
    string model = "Mustang";
};

int main() {
  Car myCar;
  myCar.honk(); // Működik! (Örökölte a Vehicle-től)
  cout << myCar.brand + " " + myCar.model; // Ford Mustang
  return 0;
}

```

---

### Az öröklődés típusai és a `protected` szint

Az öröklődés módja határozza meg, hogy a szülő tagjai milyen szinten lesznek elérhetőek a gyerekben:

| Öröklődés módja | Szülő `public` tagja | Szülő `protected` tagja | Szülő `private` tagja |
| --- | --- | --- | --- |
| **`public`** | `public` marad | `protected` marad | **Elérhetetlen** |
| **`protected`** | `protected` lesz | `protected` lesz | **Elérhetetlen** |
| **`private`** | `private` lesz | `private` lesz | **Elérhetetlen** |

> **A `protected` lényege:** Kívülről továbbra sem látható (biztonságos), de a gyerek osztályok (az örökösök) hozzáférhetnek. Ha a szülőben valami `private`, azt a gyerek sem látja! A protected öröklődés akkor jó, ha azt akarod, hogy a gyerek osztályod (és az ő majdani gyerekei) még használhassák a szülő tulajdonságait, de a külvilág felé teljesen el akarod rejteni azokat.

---

### Konstruktorok és destruktorok sorrendje

Amikor egy gyerek osztályt példányosítasz, a folyamat egy meghatározott sorrendben zajlik:

1. **Létrehozáskor:** Előbb fut le a **Szülő** konstruktora (megépül az alap), utána a **Gyereké** (rákerülnek az extrák).
2. **Megszűnéskor:** Pont fordítva! Előbb a **Gyerek** destruktora fut le, utána a **Szülőé**.

**Paraméteres konstruktor hívása:**

Ha nincsenek alapértelmezett (paraméter nélküli) konstruktorok, akkor a C++ kényszerít téged arra, hogy a gyerek osztály konstruktorában **explicit módon** megmondd, melyik szülő konstruktort és milyen adatokkal hívod meg.

Ezt az **inicializáló listában** (a `:` karakter után) kell megtenned. Ha ezt elfelejted, a fordító hibát dob, mert nem tudja "magától" létrehozni a szülő objektumot.

**Példa: Paraméteres konstruktorok láncolata**

```cpp
#include <iostream>
#include <string>
using namespace std;

// Base class
class Person {
protected:
    string name;
public:
    // Nincs default konstruktor! Csak egy paraméteres.
    Person(string n) : name(n) {
        cout << "Person constructor: name set to " << name << endl;
    }
};

// Derived class
class Employee : public Person {
protected:
    int id;
public:
    // Meg KELL hívni a Person konstruktorát az inicializáló listában
    Employee(string n, int i) : Person(n), id(i) {
        cout << "Employee constructor: id set to " << id << endl;
    }
};

// Further derived class
class Developer : public Employee {
private:
    string language;
public:
    // Itt csak a közvetlen szülőt (Employee) hívjuk meg
    Developer(string n, int i, string lang) : Employee(n, i), language(lang) {
        cout << "Developer constructor: language set to " << language << endl;
    }

    void showDetails() {
        cout << "Name: " << name << ", ID: " << id << ", Lang: " << language << endl;
    }
};

int main() {
    // A Developer létrehozása elindítja a láncot:
    // Developer -> Employee -> Person
    Developer dev("Alice", 101, "C++");
    
    dev.showDetails();
    return 0;
}

```

---

### Mi történik itt pontosan?

1. **Láncolás:** Amikor a `main`-ben meghívod a `Developer("Alice", 101, "C++")` konstruktort, az nem kezdi el rögtön a saját kódját futtatni.
2. **Átadás:** Az inicializáló listában (`: Employee(n, i)`) rögtön "felpasszolja" a labdát a közvetlen szülőnek.
3. **Továbbítás:** Az `Employee` konstruktora megkapja az `n` és `i` értékeket, de mivel neki is van szülője, ő is továbbpasszolja a nevet a `Person(n)` hívással.
4. **Végrehajtás:** A lánc legvégén a `Person` konstruktora fut le először, majd ahogy térünk vissza a hívási sorrendben, lefut az `Employee`, és legvégül a `Developer` törzse.

**Példa a sorrendről:**

```cpp
#include <iostream>
using namespace std;

// Base class (Nagyszülő)
class Grandparent {
public:
    Grandparent() {
        cout << "1. Grandparent constructor called" << endl;
    }
    ~Grandparent() {
        cout << "4. Grandparent destructor called" << endl;
    }
};

// Derived class (Szülő)
class Parent : public Grandparent {
public:
    Parent() {
        cout << "2. Parent constructor called" << endl;
    }
    ~Parent() {
        cout << "3. Parent destructor called" << endl;
    }
};

// Derived class (Gyerek)
class Child : public Parent {
public:
    Child() {
        cout << "3. Child constructor called" << endl;
    }
    ~Child() {
        cout << "2. Child destructor called" << endl;
    }
};

int main() {
    cout << "--- Creating Child object ---" << endl;
    {
        Child obj;
    } // Az objektum itt megszűnik, lefutnak a destruktorok
    cout << "--- Object destroyed ---" << endl;

    return 0;
}
```

**Miért így működik?**
* **Konstruktor (Alulról felfelé az öröklési fán):** Képzeld el, hogy egy házat építesz. Először az alapozás kell (Grandparent), utána a falak (Parent), és legvégül a tető és a dekoráció (Child). A gyerek osztály használhatja a szülő változóit, ezért a szülőnek már léteznie kell a memóriában, mire a gyerek sorra kerül.

* **Destruktor (Fentről lefelé):** A bontás fordított sorrendben történik. Előbb leszeded a díszítést (Child), aztán lebontod a falakat (Parent), és legvégül felszeded az alapot (Grandparent). Ez biztonságos, mert ha a szülőt rombolnád le először, a gyerek osztály destruktora már nem férne hozzá azokhoz az adatokhoz, amiket esetleg még használnia kellene a takarításhoz.

### Többszintű és Többszörös Öröklődés

* **Többszintű (Multilevel):** Az öröklődés láncolható.
`Nagyszülő -> Szülő -> Unoka` (Az unoka mindent lát a tetejéig).
* **Többszörös (Multiple):** Egy osztálynak több közvetlen szülője is lehet.
```cpp
class MyChild : public Father, public Mother { ... };

```

* **Veszély (Gyémánt-probléma):** Ha a `Father` és `Mother` osztálynak is ugyanaz az őse (pl. a `Grandpa`), akkor a `MyChild` két példányt kapna a nagyszülő adataiból, ami kód-ütközéshez vezet.

**Konstruktor**
Itt egy osztálynak több közvetlen szülője van. A konstruktorban az inicializáló listán mindegyik szülőt meg kell hívni, ha nincs alapértelmezett konstruktoruk.

**Példa: Többszörös öröklődés default konstruktorok nélkül**

```cpp
#include <iostream>
#include <string>
using namespace std;

class Phone {
public:
    Phone(string number) {
        cout << "Phone initialized. Number: " << number << endl;
    }
};

class Camera {
public:
    Camera(int megapixels) {
        cout << "Camera initialized: " << megapixels << " MP." << endl;
    }
};

// Okostelefon, ami Telefon ÉS Kamera is egyben
class Smartphone : public Phone, public Camera {
public:
    Smartphone(string num, int mp, string model) 
        : Phone(num), Camera(mp) { // Mindkét szülőt inicializálni kell!
        cout << "Smartphone model " << model << " is ready." << endl;
    }
};

int main() {
    Smartphone myPhone("06301234567", 108, "Galaxy S24");
    return 0;
}
```

**Példa: Többszörös öröklődés default konstruktorokkal**

```cpp
#include <iostream>
using namespace std;

// Első szülő osztály
class Light {
public:
    Light() { // Alapértelmezett konstruktor
        cout << "Light system initialized (Default)." << endl;
    }
    void turnOn() { cout << "Lights are ON." << endl; }
};

// Második szülő osztály
class Thermostat {
public:
    Thermostat() { // Alapértelmezett konstruktor
        cout << "Thermostat system initialized (Default)." << endl;
    }
    void setTemp(int t) { cout << "Temperature set to " << t << " degrees." << endl; }
};

// Gyerek osztály - több szülőtől örököl
class SmartHub : public Light, public Thermostat {
public:
    // Mivel mindkét szülőnek van default konstruktora, 
    // nem kell őket külön nevesíteni az inicializáló listában.
    SmartHub() {
        cout << "SmartHub central unit is ready." << endl;
    }
};

int main() {
    // Az objektum létrehozásakor automatikusan lefut: 
    // Light() -> Thermostat() -> SmartHub()
    SmartHub myHome;

    myHome.turnOn();
    myHome.setTemp(22);

    return 0;
}
```

### Funkciók elfedése (Function Overriding alapok)

A gyerek osztályban létrehozhatsz ugyanolyan nevű függvényt, mint ami a szülőben van. Ekkor a gyerek verziója lesz az érvényes.

```cpp
class Parent {
  public:
    void greet() { cout << "Szülő üdvözöl!"; }
};

class Child : public Parent {
  public:
    void greet() { cout << "Gyerek üdvözöl!"; } // Elfedés
};

```
