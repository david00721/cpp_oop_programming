# Fájlok

## 1. A felosztás logikája (Header vs. Source)

A C++-ban két fő fájltípussal dolgozunk, amikor osztályokat vagy függvényeket emelünk ki:

* **Header fájl (`.h` vagy `.hpp`): A "Mi ez?" (Interfész)**
* Ide kerülnek a deklarációk: az osztályok felépítése, a tagváltozók, és a tagfüggvények szignatúrái (vagyis a nevük, visszatérési értékük és paramétereik).
* Ezt a fájlt fogják `include`-olni más fájlok, amik használni akarják ezt az osztályt.


* **Source fájl (`.cpp`): A "Hogyan működik?" (Implementáció)**
* Ide kerül a tényleges kód, a függvények törzse.
* Ez a fájl `include`-olja a saját header fájlját, hogy lássa az osztály szerkezetét, majd definiálja a benne lévő metódusokat a hatókör operátor (`::`) segítségével.

---

## 2. Többszörös beillesztés elleni védelem (Include Guards)

Ha egy header fájlt több különböző `.cpp` fájl is beilleszt (ami teljesen normális), a fordító könnyen abba a hibába eshet, hogy kétszer próbálja meg létrehozni ugyanazt az osztályt. Ennek elkerülésére **kötelező** védelmet tenni minden header fájlba.

**A modern és legegyszerűbb megoldás:**

```cpp
#pragma once // A fájl legelső sora

```

Ez megmondja a fordítónak, hogy ezt a fájlt csak egyszer szabad beolvasni a fordítás során.

**A klasszikus megoldás (ha nagyon régi fordítóval dolgozol):**

```cpp
#ifndef CALCULATOR_H
#define CALCULATOR_H
// Ide jön az osztály kódja...
#endif

```

## 3. Gyakorlati Példa: Szétdarabolt Osztály

Nézzük meg egy egyszerű `Calculator` osztály szétvágását!

**1. Calculator.h (A Deklaráció)**

```cpp
#pragma once

class Calculator {
private:
    int lastResult;

public:
    // Konstruktor deklaráció
    Calculator(); 

    // Metódus deklarációk (itt nincs törzsük!)
    int add(int a, int b);
    int getLastResult() const;
};

```

**2. Calculator.cpp (A Definíció / Implementáció)**

```cpp
#include "Calculator.h" // Kötelező beilleszteni a saját headerjét!

// A Calculator:: jelzi, hogy ez a konstruktor a Calculator osztályhoz tartozik
Calculator::Calculator() : lastResult(0) {}

// A :: az úgynevezett "Scope Resolution Operator" (Hatókör operátor)
int Calculator::add(int a, int b) {
    lastResult = a + b;
    return lastResult;
}

int Calculator::getLastResult() const {
    return lastResult;
}

```

**3. main.cpp (A Felhasználás)**

```cpp
#include <iostream>
#include "Calculator.h" // Csak a headert illesztjük be! A .cpp-t SOHA.

int main() {
    Calculator calc;
    std::cout << "Eredmeny: " << calc.add(5, 10) << std::endl;
    return 0;
}

```

## 4. Mi történik a háttérben? (A Fordítási Folyamat)

A C++ egy specifikus folyamaton megy végig, amikor ebből futtatható programot csinál.

1. **Előfeldolgozó (Preprocessor):** Végigmegy a `#include` utasításokon, és a header fájlok tartalmát szövegszerűen bemásolja oda, ahonnan meghívták őket.
2. **Fordító (Compiler):** Minden `.cpp` fájlt (fordítási egységet) **külön-külön** lefordít úgynevezett Object (`.o` vagy `.obj`) gépi kódú fájlokká. A `main.cpp` fordításakor a fordító elhiszi, hogy az `add` metódus létezik (mert látta a headerben), de még nem tudja, hol van a tényleges kódja.
3. **Szerkesztő (Linker):** Fogja a különálló `.obj` fájlokat (`main.obj` és `Calculator.obj`), összeköti a "hiányzó láncszemeket" (pl. megkeresi az `add` metódus implementációját), és összerakja belőlük a végső `.exe` programot.


# Operátor overloading

### 1. Alapvető operátor kategóriák (Gyorstalpaló)

A mindennapi programozás során ezeket a beépített operátorokat használjuk a primitív típusokon (`int`, `float`, `bool`):

| Kategória | Operátorok | Példa / Leírás |
| --- | --- | --- |
| **Aritmetikai** | `+`, `-`, `*`, `/`, `%` | A `%` (modulo) az osztás maradékát adja meg (pl. `10 % 3 = 1`). |
| **Értékadó** | `=`, `+=`, `-=`, `*=`, `/=` | `a += 5` pontosan ugyanazt jelenti, mint `a = a + 5`. |
| **Relációs** | `==`, `!=`, `<`, `>`, `<=`, `>=` | Két értéket hasonlítanak össze, eredményük mindig `bool` (igaz/hamis). |
| **Logikai** | `&&`, ` |  |
| **Bitenkénti** | `&`, ` | `, `^`, `~`, `<<`, `>>` |

---

### 2. Pre-inkrementálás vs. Post-inkrementálás (`++x` vs `x++`)

Mindkét operátor megnöveli a változó értékét 1-gyel, de a **végrehajtás sorrendje és a memóriahasználat** eltérő.

* **Pre-inkrementálás (`++x`):** Először megnöveli a változó értékét, és *utána* tér vissza az új, megnövelt értékkel. Nincs szükség másolat készítésére a memóriában.
* **Post-inkrementálás (`x++`):** Először készít egy másolatot a jelenlegi értékről, megnöveli az eredeti változót, majd visszatér a *régi érték másolatával*.

```cpp
int a = 5;
int b = ++a; // PRE: 'a' azonnal 6 lesz, 'b' is megkapja a 6-ot.

int c = 5;
int d = c++; // POST: 'd' megkapja a régi értéket (5), és csak utána lesz 'c' értéke 6.

```

> **Mérnöki ökölszabály:** C++-ban (különösen iterátoroknál és bonyolult objektumoknál) **mindig a `++x` (pre-inkrementálás) formát használd** a `for` ciklusokban, mert a post-inkrementálás felesleges memóriamásolást végez a háttérben, ami lassítja a programot.

### 3. Operátorok túlterhelése (Operator Overloading) osztályokban

A C++ lehetővé teszi, hogy a beépített operátorokat (pl. `+`, `==`, `<<`) megtanítsd arra, hogyan működjenek a te saját, egyedi osztályaiddal. Ez nem más, mint egy speciális tagfüggvény írása, aminek a neve `operator` és a használni kívánt jel.

Itt tökéletesen összeér minden, amit eddig tanultunk: **konstans tagfüggvények** és **konstans referenciák** kellenek a biztonságos és gyors működéshez.

Nézzünk egy 2D pontot (`Point`) reprezentáló osztályt, ahol meg akarunk adni egy összeadás (`+`) és egy egyenlőségvizsgálat (`==`) operátort:

```cpp
#pragma once

class Point {
private:
    int x;
    int y;

public:
    // Konstruktor
    Point(int xCoord, int yCoord) : x(xCoord), y(yCoord) {}

    // 1. Az összeadás (+) operátor túlterhelése
    // - Visszatérési érték: Egy új Point objektum (érték szerint, mert új objektum születik).
    // - Paraméter: const Point& (nem másoljuk le a másik pontot, és nem is módosítjuk).
    // - Tagfüggvény vége: const (az eredeti pontot sem módosítjuk, csak olvasunk belőle).
    Point operator+(const Point& other) const {
        return Point(this->x + other.x, this->y + other.y);
    }

    // 2. Az egyenlőség (==) operátor túlterhelése
    // - Visszatérési érték: bool (igaz, ha a koordináták megegyeznek).
    bool operator==(const Point& other) const {
        return (this->x == other.x && this->y == other.y);
    }

    int getX() const { return x; }
    int getY() const { return y; }
};

```

**Így néz ki a felhasználása a `.cpp` fájlban:**

```cpp
#include <iostream>
#include "Point.h"

int main() {
    Point p1(10, 20);
    Point p2(5, 5);

    // Itt a háttérben valójában ez hívódik meg: p1.operator+(p2)
    Point p3 = p1 + p2; 

    std::cout << "p3 koordinatak: X=" << p3.getX() << ", Y=" << p3.getY() << std::endl;

    // Itt pedig: p1.operator==(p2)
    if (p1 == p2) {
        std::cout << "A ket pont megegyezik." << std::endl;
    } else {
        std::cout << "A ket pont kulonbozik." << std::endl;
    }

    return 0;
}

```
