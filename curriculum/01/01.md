# Osztály alapok

## 1. Struct vs. Class C++-ban

A C++-ban már ismert `struct` egy felhasználó által definiált típus, amely változókat és függvényeket tartalmaz.

### A Struct problémája

A struktúrák legnagyobb hátulütője, hogy alapértelmezett esetben **nem valósítják meg az adatrejtést**.

* Minden tagváltozó publikus (`public`), tehát kívülről bárki módosíthatja őket.
* Ez inkonzisztens állapotokhoz vezethet (pl. egy dátum struktúránál `d = 32`).

### A Class (Osztály) megoldása

Az osztályok szintén felhasználói típusok, de saját névteret alkotnak és szigorúbb hozzáférési szabályokkal rendelkeznek.
* **Adatrejtés:** Alapértelmezett esetben az osztály minden tagja privát (`private`), ellentétben a structtal.
* A „kívülről” elérhető függvényeket `public`, míg a belső működést `private` vagy `protected` tagokkal valósítják meg.

#### Példakód (Date osztály):

```cpp
class Date {
    int d, m, y; // default: minden private [cite: 38]
public:
    void init(int dd, int mm, int yy);
    void print();
};
```

## 2. Metódusok definiálása és szeparálása

Egy osztály metódusait kétféleképpen definiálhatjuk:

### A. Osztályon belül (Inline)

Ha a metódus törzse nagyon rövid, gyakran a deklarációnál (a `.h` fájlban) definiálják.  
Ekkor a fordító „inline-olhatja”, azaz beilleszti a kódba a hívás helyére, megspórolva a függvényhívás költségét.

```cpp
class Date {
    int d, m, y;
public:
    void print() { // belül definiált metódus
        std::cout << d << "/" << m << "/" << y << std::endl;
    }
};
```

### B. Osztályon kívül (.cpp fájlban)

A deklarációt általában a header fájlba (`.h`), a definíciót pedig külön fordítási egységbe (`.cpp`) tesszük.

**Miért jó ez?**  
Ha változik az implementáció, csak azt az egy `.cpp` fájlt kell újrafordítani, nem az egész programot.

**Fontos:** Mivel az osztály saját névteret képez, a definíciónál használni kell az osztály nevét (pl. `Date::`).

```cpp
// .cpp fájl
void Date::init(int dd, int mm, int yy) { // kívül definiált metódus [cite: 66]
    d = dd;
    m = mm;
    y = yy;
}
```

**Gyakorló feladat:**  
Írj egy `Circle` (Kör) osztályt!  
Legyen egy privát `radius` (sugár) változója, és egy publikus `setRadius(int r)` függvénye, amit az osztályon kívül definiálsz meg!

## 3. Inicializálás: A Konstruktor

A fenti `init()` függvénnyel van egy nagy baj: a programozó elfelejtheti meghívni.  
Ekkor a változók memóriaszemetet tartalmazhatnak. Erre megoldás a konstruktor.

### A konstruktor jellemzői

- A neve megegyezik az osztály nevével.
- Nincs visszatérési értéke (még `void` sem!).
- Akárhány argumentuma lehet (több konstruktor is létezhet, ha különböznek a paramétereikben).

### Típusai

**Default konstruktor:**  
Argumentum nélküli konstruktor. Ha nem írunk egyetlen konstruktort sem, a fordító automatikusan generál egyet (de ez nem biztos, hogy jól inicializál).

**User-defined konstruktor:**  
Ha legalább egy konstruktort írunk, a fordító már nem generál automatikus default konstruktort.

### Példa konstruktorral

```cpp
class Date {
    int d, m, y;
public:
    // Paraméteres konstruktor deklarációja
    Date(int dd, int mm, int yy); 

    // Default konstruktor (ha üresen hozzuk létre)
    Date() { d = 1; m = 1; y = 1970; } 
};
```

```cpp
// Implementáció kívül
Date::Date(int dd, int mm, int yy) {
    d = dd; m = mm; y = yy;
}
```

```cpp
// Használat:
Date mydate(23, 7, 1985); // Paraméteres hívás [cite: 206]
Date mydate2;             // Default hívás [cite: 207]
```

## Gyakorló feladatok

### 1. Alapok (Struktúrából Osztály)

Készíts egy `Book` nevű osztályt!

- Legyenek privát adattagjai: `title` (string), `pages` (int).
- Legyen egy publikus `setup(string t, int p)` függvénye, ami beállítja ezeket az értékeket.
- Legyen egy publikus `display()` függvénye, ami kiírja az adatokat.

**Cél:** Megérteni a `private` / `public` kulcsszavakat és az egyszerű tagfüggvényeket.

### 2. Konstruktorok bevezetése

Bővítsd az előző `Book` osztályt!

- Töröld a `setup` függvényt.
- Helyette írj egy konstruktort, ami paraméterként várja a címet és az oldalszámot, és beállítja azokat.

```cpp
Book myBook("Harry Potter", 300);
```

### 3. Default konstruktor és Inline

Készíts egy `Point` (Pont) osztályt, ami egy 2D koordinátát tárol (`x`, `y`).

- Írj egy paraméter nélküli (default) konstruktort, ami az `x = 0`, `y = 0` értékeket állítja be.  
  Ezt írd meg inline (az osztályon belül)!
- Írj egy kétparaméteres konstruktort is `Point(int x, int y)` néven.

A `main`-ben hozz létre egy pontot az origóban (default) és egyet az `(5, 10)` helyen.

### 4. Definíció és Deklaráció szétválasztása

Készíts egy `BankAccount` (Bankszámla) osztályt!

- Adattag: `balance` (egyenleg, `double`)
- Konstruktor: állítsa be a kezdő egyenleget
- Metódusok:
  - `deposit(double amount)` (befizetés)
  - `withdraw(double amount)` (kivét)

**Feladat:**  
Az osztály deklarációja maradjon csak meg, a metódusok kifejtését (implementációját) a  
`class BankAccount { ... };` blokkon kívül végezd el a `::` operátorral!

### 5. Komplexebb logika és Validáció

Készíts egy `Triangle` (Háromszög) osztályt!

- Adattagok: `a`, `b`, `c` oldalak hossza (`int`)
- Konstruktor: várja a három oldalt  
  **Fontos:** A konstruktorban ellenőrizd, hogy a megadott számok pozitívak-e!  
  Ha valamelyik nem pozitív, állítsd mindet alapértelmezetten `1`-re.
- Metódus: `isEquilateral()` – térjen vissza `true`-val, ha a háromszög szabályos (minden oldala egyenlő), egyébként `false`.

A `main` függvényben teszteld le helyes és helytelen adatokkal is.
