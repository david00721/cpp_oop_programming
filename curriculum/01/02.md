## 1. Getterek és Setterek (Adatrejtés II.)

Az előző órán tanultuk, hogy az osztályok adattagjait `private` láthatósággal védjük.  
De hogyan érjük el őket biztonságosan?

- **Getter (Lekérdező):** Egy publikus függvény, ami visszaadja a privát változó értékét.
- **Setter (Beállító):** Egy publikus függvény, ami beállítja a privát változó értékét (gyakran ellenőrzéssel).

### Példa

```cpp
class Node {
    int value; // privát
public:
    Node(int v) : value(v) {} // Konstruktor (inicializáló listával, lásd később)

    // Getter - const, mert nem módosít (lásd 3. pont)
    int getValue() const { return value; }

    // Setter
    void setValue(int v) { value = v; }
};
```

## 2. Láncolt Hívások (Method Chaining)

Gyakori igény, hogy több beállítást egymás után fűzzünk:

```cpp
obj.setValue(5).setNext(ptr).setVisibility(true);
```

Ehhez a settereknek nem `void`-dal kell visszatérniük, hanem az objektumra mutató referenciával.

- A `this` pointer az aktuális objektumra mutat
- A `*this` maga az objektum
- A visszatérési típus: `OsztályNeve&`

### Helyes megvalósítás

```cpp
class Node {
    int value;
    bool visible;
public:
    Node& setValue(int v) { 
        value = v; 
        return *this; 
    }

    Node& setVisible(bool b) { 
        visible = b; 
        return *this; 
    }
};
```

```cpp
// Használat:
// Node n;
// n.setValue(10).setVisible(true);
```

## 3. A `const` kulcsszó használata

A `const` azt jelenti: *„megígérem, hogy ez az érték nem változik”*.  
A C++-ban több helyen is használjuk.

### A. Konstans változók

Létrehozáskor kötelező inicializálni őket, később nem módosíthatók.

```cpp
const int a = 5; // OK
// const int b;  // HIBA! Nincs kezdőérték.
// a = 6;        // HIBA! Nem módosítható.
```

### B. Konstans tagváltozók és az inicializáló lista

Ha egy osztálynak `const` adattagja van, azt **csak az inicializáló listában** lehet beállítani.

```cpp
class Data {
    const int id; // Ezt egyszer beállítjuk, utána örök
public:
    Data(int i) : id(i) { 
        // id = i; // Ez itt már HIBA lenne!
    }
};
```

### C. Konstans tagfüggvények

Ha egy metódus nem módosítja az osztály állapotát, ki kell tenni a `const` jelzőt a függvény végére.

**Szabály:** `const` objektumon csak `const` metódus hívható meg!

```cpp
int getValue() const { return value; }
```

**Mutable:**  
Ha egy változót `const` függvényben is módosítani akarunk (pl. cache), `mutable` kulcsszót használunk.

### D. Konstans pointerek (trükkös!)

Olvasd **hátulról előre**:

- `const int* p`  
  Pointer egy konstans int-re (az érték nem módosítható, a pointer igen).

- `int* const p`  
  Konstans pointer egy int-re (a pointer nem változhat, az érték igen).

## 4. Referenciák

A referencia egy alternatív név (alias) egy változóhoz.

- Úgy viselkedik, mint egy konstans pointer
- Nem lehet `nullptr`
- Kötelező inicializálni

### Mire jó?

- **Hatékonyság:** nem másolódik le az objektum  
  → `const T&`
- **Kimeneti paraméter:** módosíthatja a hívó változóját

```cpp
void print(const std::string& text); // Gyors, nem másol, nem módosít
void update(int& score);             // Módosítja az eredetit
```

**Vigyázat:**  
Soha ne adj vissza referenciát lokális változóra! (Dangling reference)

## 5 db Gyakorló Feladat (Könnyűtől a Nehézig)

### 1. Szint: Getter / Setter és Inicializáló Lista

Készíts egy `Employee` osztályt!

- Adattagok: `const int id`, `int salary`
- Konstruktor: várja az `id`-t és a `salary`-t  
  *(id csak inicializáló listában!)*
- Metódusok:
  - `getId()` – const
  - `setSalary(int s)`

---

### 2. Szint: Konstans Pointerek értelmezése

- `int x = 10;`, `int y = 20;`
- `const int*` – pointer átállítható, érték nem
- `int* const` – pointer nem állítható, érték igen

**Cél:** Megérteni a `const` helyét a deklarációban.

---

### 3. Szint: Láncolt hívások

Készíts egy `Printer` osztályt!

- Adattagok: `string text`, `string color`
- Metódusok:
  - `setText(string t)`
  - `setColor(string c)`
  - `print()`

A setterek `Printer&`-rel térjenek vissza.

```cpp
myPrinter.setText("Hello").setColor("Red").print();
```

---

### 4. Szint: Referencia vs Másolás

Készíts egy `VectorWrapper` osztályt (`std::vector<int>`).

- `getSize()`
- `add(int x)`

Írj:
- `processVector(VectorWrapper v)`
- `processVectorRef(VectorWrapper& v)`

Írd le kommentben, miért hatékonyabb a referencia (copy constructor).

---

### 5. Szint: Mutable cache (Mester feladat)

Készíts egy `ComplexCalculation` osztályt!

- `int input`
- `mutable int cachedResult`
- `mutable bool isCalculated`

- Konstruktor: `isCalculated = false`
- `getResult() const`

Ha még nincs kiszámolva → számol, cache-el  
Ha már igen → cache-t adja vissza

**Cél:** `mutable` használata technikai okból (caching).
