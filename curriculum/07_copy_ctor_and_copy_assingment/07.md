### 1. A Másoló Konstruktor (Copy Constructor)

**Mi ez?** Egy speciális konstruktor, ami egy **teljesen új objektumot** hoz létre úgy, hogy lemásolja egy már meglévő objektum adatait.
**Mikor hívódik meg?** Kizárólag az objektum *megszületésének pillanatában*.

Három tipikus esete van:

1. Amikor egy új objektumot egy meglévővel inicializálsz: `Player p2 = p1;` vagy `Player p2(p1);`
2. Amikor egy objektumot érték szerint adsz át egy függvénynek: `void doSomething(Player p)`
3. Amikor egy függvény egy objektummal tér vissza (bár ezt a modern fordítók gyakran optimalizálják).

**Szintaxis:** Paraméterként egy konstans referenciát vár a saját típusából (`const ClassName&`).

```cpp
class Player {
private:
    int health;
    int score;

public:
    // Sima paraméteres konstruktor
    Player(int h, int s) : health(h), score(s) {}

    // 1. MÁSOLÓ KONSTRUKTOR
    // Egy meglevő 'other' játékos adataival hozzuk létre ezt az új játékost
    Player(const Player& other) : health(other.health), score(other.score) {
        std::cout << "--> Masolo Konstruktor lefutott!" << std::endl;
    }
};

```

### 2. A Másoló Értékadó Operátor (Copy Assignment Operator)

**Mi ez?** Egy operátor túlterhelés, ami felülírja egy **már létező** objektum adatait egy másik, szintén létező objektum adataival.
**Mikor hívódik meg?** Amikor mindkét objektum már korábban létrejött, és egyenlőségjelet teszel közéjük: `p2 = p1;`

**Szintaxis:** Szintén konstans referenciát vár, de a visszatérési értéke egy referencia saját magára (`ClassName&`), hogy az értékadásokat láncolni lehessen (pl. `p1 = p2 = p3;`).

```cpp
class Player {
private:
    int health;
    int score;

public:
    Player(int h, int s) : health(h), score(s) {}

    // 2. MÁSOLÓ ÉRTÉKADÓ OPERÁTOR
    Player& operator=(const Player& other) {
        std::cout << "--> Masolo Ertekado Operator lefutott!" << std::endl;
        
        // Biztonsági ellenőrzés: Ön-értékadás kiszűrése (pl. ha valaki ezt írja: p1 = p1;)
        // Összehasonlítjuk a memóriacímeket. Ha megegyeznek, nem csinálunk semmit.
        if (this == &other) {
            return *this; 
        }

        // Adatok felülírása
        this->health = other.health;
        this->score = other.score;

        // Visszatérünk magával az objektummal
        return *this;
    }
};

```

### Lássuk a különbséget a gyakorlatban!

Ez az a rész, ami a leggyakrabban megzavarja a kezdőket, mert mindkét esetben az `=` jelet használjuk, de a fordító mást hív meg a háttérben:

```cpp
int main() {
    // Sima konstruktor hívódik meg
    Player p1(100, 5000); 

    // ESET A: MÁSOLÓ KONSTRUKTOR
    // Itt a 'p2' most születik meg. Az egyenlőségjel ellenére ez inicializálás!
    Player p2 = p1;  // (Ugyanaz, mintha ezt írnád: Player p2(p1); )

    // Sima konstruktor hívódik meg a 'p3'-ra is
    Player p3(50, 1000);

    // ESET B: MÁSOLÓ ÉRTÉKADÓ OPERÁTOR
    // Itt a 'p3' és a 'p2' már léteznek. A 'p3' meglévő adatait felülírjuk a 'p2'-vel.
    p3 = p2; 

    return 0;
}

```

### Beépített másolás

***Tudod, mi a legszebb ebben?***
Ha a fenti `Player` osztályból **teljesen kitörölnéd** a Másoló konstruktort és az Értékadó operátort, a kódod akkor is hibátlanul működne és lefordulna!

***Miért?***
Mert egyszerű, primitív adattagoknál (`int`, `bool`, `float`) a C++ fordító **automatikusan generál** neked egy Másoló konstruktort és egy Értékadó operátort, amik pontosan ezt csinálják: fogják a változókat, és egyenként átmásolják az értéküket (ezt hívjuk *sekély másolásnak*).

***Akkor miért tanultuk most meg ezeket manuálisan megírni?***
Azért, mert abban a pillanatban, hogy az osztályodba bekerül egy **mutató (pointer)** vagy egy dinamikus tömb, a fordító által generált automatikus megoldás már nem lesz jó, és nekünk kell átvenni az irányítást.

Kiváló, akkor pontosan innen folytatjuk! Nézzük meg, mi történik, ha "felrobbantjuk" a fordító kényelmes beépített másolását azzal, hogy behozunk egy mutatót és a dinamikus memóriakezelést.

### Mutatók és a Sekély Másolás (Shallow Copy)

Alakítsuk át a `Player` osztályunkat! Tegyük fel, hogy az életerő (`health`) marad egy sima `int`, de a tapasztalati pontokat (`xp`) mostantól egy dinamikusan lefoglalt memóriaterületen tároljuk, egy mutató (`int*`) segítségével. Mivel használunk `new` operátort a konstruktorban, kötelező írnunk egy Destruktort is (`~Player`), ami a `delete` segítségével takarít az objektum megsemmisülésekor.

Ha most **nem** írjuk meg a saját másoló függvényeinket, a fordító a szokásos "sekély másolást" (shallow copy) végzi el: fogja a mutatót, és lemásolja magát a memóriacímet.

**Az eredmény katasztrofális lesz:**

1. A két játékos (`p1` és `p2`) ugyanarra a memóriaterületre fog mutatni. Ha `p2` szerez egy kis XP-t, `p1` XP-je is varázslatosan megnő.
2. Amikor a program véget ér, `p1` megsemmisül, és a destruktora felszabadítja a memóriát. Majd `p2` is megsemmisül, és megpróbálja felszabadítani *ugyanazt* a már törölt memóriát. Ez az úgynevezett **Double Free** hiba, ami azonnal összeomlasztja a programot.

### A Megoldás: Mély Másolás (Deep Copy)

Ahhoz, hogy ezt elkerüljük, át kell vennünk az irányítást. Meg kell írnunk a Másoló Konstruktort és az Értékadó Operátort úgy, hogy ne csak a címet másolják le, hanem **foglaljanak egy teljesen új memóriaterületet**, és oda másolják be az eredeti *értéket*. Ezt hívjuk mély másolásnak.

Íme a "pointeresített", hibátlanul működő `Player` osztály:

```cpp
#include <iostream>

class Player {
private:
    int health;
    int* xp; // Mutató egy dinamikusan foglalt értékre!

public:
    // Sima paraméteres konstruktor (memóriát foglal)
    Player(int h, int startingXp) : health(h) {
        xp = new int(startingXp); // Új memóriaterület lefoglalása
    }

    // DESTRUKTOR (Takarít az objektum megsemmisülésekor)
    ~Player() {
        delete xp; 
    }

    // 1. MÁSOLÓ KONSTRUKTOR (Mély másolás!)
    Player(const Player& other) : health(other.health) {
        std::cout << "--> Masolo Konstruktor (Mely masolas) lefutott!" << std::endl;
        // NEM a címet másoljuk! Új memóriát foglalunk, és a kapott ÉRTÉKET (*other.xp) tesszük bele.
        xp = new int(*(other.xp)); 
    }

    // 2. MÁSOLÓ ÉRTÉKADÓ OPERÁTOR (Mély másolás!)
    Player& operator=(const Player& other) {
        std::cout << "--> Masolo Ertekado Operator (Mely masolas) lefutott!" << std::endl;
        
        // Biztonsági ellenőrzés (Ön-értékadás kiszűrése)
        if (this == &other) {
            return *this; 
        }

        // Sima adatok felülírása
        this->health = other.health;

        // Pointeres adatok felülírása (Kritikus 3 lépés!)
        // Lépés 1: Eltakarítjuk a SAJÁT régi memóriánkat, hogy ne legyen memóriaszivárgás
        delete this->xp; 
        
        // Lépés 2: Új memóriát foglalunk
        this->xp = new int;
        
        // Lépés 3: Átmásoljuk az értéket
        *(this->xp) = *(other.xp);

        // Visszatérünk magával az objektummal
        return *this;
    }

    // Segédmetódus a teszteléshez
    void print() const {
        std::cout << "Health: " << health << ", XP: " << *xp << std::endl;
    }
    
    // XP módosítása
    void setXp(int newXp) {
        *xp = newXp;
    }
};

int main() {
    Player p1(100, 5000); 

    // ESET A: MÁSOLÓ KONSTRUKTOR (Új memória jön létre a p2 számára)
    Player p2 = p1;  

    // ESET B: MÁSOLÓ ÉRTÉKADÓ OPERÁTOR (A p3 régi memóriája törlődik, újat kap)
    Player p3(50, 1000);
    p3 = p2; 

    // Teszteljük le a függetlenséget!
    // Ha átírjuk p2 XP-jét, a p1 és p3 értéke NEM változhat meg!
    p2.setXp(9999);

    std::cout << "--- Eredmenyek ---" << std::endl;
    p1.print(); // XP: 5000 marad
    p2.print(); // XP: 9999 lett
    p3.print(); // XP: 5000 marad

    // A program végén 3-szor fut le a Destruktor. 
    // Mivel mind a 3 objektumnak saját dinamikus memóriája van, a program hibátlanul leáll.
    return 0;
}

```

### A Hármas Szabály (The Rule of Three)

Ezzel a lépéssel a C++ egyik legfontosabb objektumorientált alapelvéhez érkeztünk. Amit most fent megírtunk, az a híres Hármas Szabály.

**A szabály kimondja:**
Ha egy osztályban szükséged van a következő három speciális tagfüggvény *bármelyikére* (mert például memóriát foglalsz), akkor szinte biztosan szükséged van **mind a háromra**:

1. Destruktor
2. Másoló Konstruktor
3. Másoló Értékadó Operátor
